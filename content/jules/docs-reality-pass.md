# Routine: Docs Reality Pass

**Goal:** Reconcile documentation with the actual shipped behavior of the codebase.

## Task Details
1. **Find One Contradiction:** Identify one place where docs, roadmap notes, or to-do content misstate the current state of the project.
2. **Correct the Record:** Update the relevant documentation so it reflects reality.
3. **Preserve Forward Motion:** If the item is partially complete, rewrite it as a next-step item instead of deleting it entirely.
4. **Tighten Wording:** Remove stale claims, duplicated future work, or misleading phrasing that could confuse future sessions.

## Execution Reminders
* Prioritize `README.md`, meta roadmap content, and aspirational docs.
* Focus on accuracy over flourish.
* Keep edits small and explicit.
* **Upon successful completion, you MUST write a short log (including date, routine name, success status, and any learnings or suggestions for improving this routine) to a new markdown file in `content/jules/reports/` (e.g., `content/jules/reports/[date]-[routine-name].md`).**
