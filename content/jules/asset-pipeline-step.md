# Routine: Asset Pipeline Step

**Goal:** Make one incremental improvement toward a dependable asset pipeline for static output.

## Task Details
1. **Choose One Improvement:** Implement one bounded enhancement related to asset handling.
   - Examples: copying assets into output, defining asset directory behavior, preserving directory structure, or documenting expected asset flow.
2. **Stay Narrow:** Do not attempt hashing, optimization, and pipeline redesign all at once.
3. **Verify Output:** Ensure the selected asset behavior is testable and reflected in generated output.
4. **Add Coverage:** Create or extend a fixture or test that proves the asset behavior works as intended.

## Execution Reminders
* Preserve simple mental models.
* Avoid unnecessary dependencies.
* Note any breaking implications for output structure.
* **Upon successful completion, you MUST write a short log (including date, routine name, success status, and any learnings or suggestions for improving this routine) to a new markdown file in `content/jules/reports/` (e.g., `content/jules/reports/[date]-[routine-name].md`).**
