# Routine: Taxonomy Step

**Goal:** Add one small piece of taxonomy or tagging support using frontmatter.

## Task Details
1. **Choose One Narrow Capability:** Examples include parsing `tags`, storing them in metadata, or generating one simple tag index page.
2. **Keep Scope Tight:** Do not implement the entire taxonomy system in one task.
3. **Fit Existing Frontmatter Behavior:** Extend current metadata handling instead of replacing it.
4. **Verify with Real Content:** Add or update sample content that exercises the new taxonomy behavior.

## Execution Reminders
* Favor a simple data model.
* Avoid overengineering around future taxonomy features.
* Make the first step easy to build on later.
* **Upon successful completion, you MUST write a short log (including date, routine name, success status, and any learnings or suggestions for improving this routine) to a new markdown file in `content/jules/reports/` (e.g., `content/jules/reports/[date]-[routine-name].md`).**
