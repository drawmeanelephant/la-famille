# Routine: Close One Stub

**Goal:** Replace one generated missing page with a real documentation page that improves the usability of the site.

## Task Details
1. **Pick One Missing Page:** Find one linked documentation page that currently resolves to a generated stub.
2. **Write the Real Page:** Create the missing markdown file in the appropriate `content/` location.
3. **Keep It Useful:** The page should be concise, accurate, and connected to the current system as implemented.
4. **Link Thoughtfully:** Add at least one relevant internal link so the new page participates in the content graph.
5. **Do Not Overexpand:** Complete one page well instead of starting several unfinished ones.

## Execution Reminders
* Prefer docs linked from `content/help.md` or the documentation index.
* Match the existing frontmatter and documentation tone.
* Run the build and verify the stub is replaced by a real rendered page.
* **Upon successful completion, you MUST write a short log (including date, routine name, success status, and any learnings or suggestions for improving this routine) to a new markdown file in `content/jules/reports/` (e.g., `content/jules/reports/[date]-[routine-name].md`).**
