# Routine: Metadata Feature Step

**Goal:** Add one small feature that uses generated metadata to improve navigation, discovery, or understanding of the site.

## Task Details
1. **Choose One Metadata Output:** Use `graph.json`, `backlinks.json`, or `meta.json`.
2. **Build One Improvement:** Implement one feature such as backlinks display, orphan-page reporting, basic related-pages logic, or a metadata-driven index.
3. **Keep It Lightweight:** Avoid turning this into full search or taxonomy unless the chosen step naturally fits.
4. **Verify End-to-End:** Ensure the metadata is generated correctly and the new consumer behaves as expected.

## Execution Reminders
* Reuse existing output before inventing new formats.
* Prefer visible user value or clear structural value.
* Add tests where practical.
* **Upon successful completion, you MUST write a short log (including date, routine name, success status, and any learnings or suggestions for improving this routine) to a new markdown file in `content/jules/reports/` (e.g., `content/jules/reports/[date]-[routine-name].md`).**
