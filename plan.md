# Implementation Plan - RAG Artifact Hygiene & Deployment Cleanup

## Summary
Clean up RAG generation and publication behavior by ensuring `rag-archive/` is strictly a local generated-only output directory ignored by Git, while maintaining optional publication to `public/rag-archive/` in the GitHub Pages workflow without repository write operations or git commit/push steps.

## User Review Required
> [!NOTE]
> The workflow permissions in `.github/workflows/deploy.yml` will be reduced from `contents: write` to `contents: read`. No repository modifications will occur during GitHub Pages deployment.

#### [MODIFY] [.gitignore](file:///Users/tbuddy/Documents/antigravity/la-famille/.gitignore)

### Deployment Workflow
#### [MODIFY] [.github/workflows/deploy.yml](file:///Users/tbuddy/Documents/antigravity/la-famille/.github/workflows/deploy.yml)
